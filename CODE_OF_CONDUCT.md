# Code of Conduct

## Our Commitment

The CKS Runtime project is committed to providing a welcoming, respectful, and collaborative environment for everyone.

Participation in this project should be based on professionalism, constructive discussion, and mutual respect regardless of experience, background, identity, or perspective.

---

## Expected Behavior

Contributors are encouraged to:

* communicate respectfully;
* discuss ideas rather than individuals;
* provide constructive feedback;
* assume good intentions;
* help improve the project through thoughtful collaboration;
* respect differing technical opinions.

---

## Unacceptable Behavior

The following behaviors are not acceptable within the CKS Runtime community:

* harassment or discrimination;
* personal attacks or insults;
* intimidation or threats;
* trolling or deliberately disruptive behavior;
* publishing private information without permission;
* spam or intentionally misleading contributions.

---

## Project Philosophy

CKS Runtime values:

* scientific rigor;
* openness;
* reproducibility;
* constructive criticism;
* long-term collaboration.

Technical disagreements are expected and encouraged when conducted respectfully and supported by clear reasoning.

---

## Enforcement

Project maintainers are responsible for interpreting and enforcing this Code of Conduct.

They may remove, edit, or reject contributions or comments that violate these guidelines.

Repeated or serious violations may result in temporary or permanent exclusion from project participation.

---

## Scope

This Code of Conduct applies to:

* GitHub issues;
* pull requests;
* discussions;
* documentation;
* project communications;
* all official CKS Runtime community spaces.

---

## Reporting

If you believe someone has violated this Code of Conduct, please contact the project maintainer through the repository issue tracker or GitHub profile.

All reports will be handled respectfully and confidentially whenever possible.
